# Caching

> Back to [README](../README.md)

---

bdfinance includes a pluggable caching layer that reduces redundant HTTP requests to DSE. Caching is **enabled by default** with an in-memory backend.

---

## Architecture

```
BDStockClient
  └── CacheManager
        └── CacheBackend (abstract)
              ├── MemoryCache   (aiocache MEMORY)
              └── RedisCache    (aiocache REDIS)
```

### CacheManager

`CacheManager` is the main cache interface used by repositories. It provides:

| Method | Description |
|--------|-------------|
| `get(prefix, **kwargs)` | Retrieve a cached value |
| `set(prefix, value, ttl=None, **kwargs)` | Store a value with TTL |
| `clear()` | Clear all cache entries |
| `close()` | Close the cache connection |
| `generate_key(prefix, **kwargs)` | Generate a deterministic cache key |

### Key Generation

Cache keys are generated by hashing a combination of a prefix string and sorted keyword arguments:

```
bdfinance:{prefix}:{md5_hash}
```

The MD5 hash is computed from JSON-serialized, sorted kwargs. This ensures that identical parameters always produce the same cache key regardless of argument order.

## Backends

### Memory (default)

Uses `aiocache` with the `MEMORY` backend and `PickleSerializer`.

```python
from bdfinance import CacheConfig

config = CacheConfig(backend="memory", ttl=300, max_size=1000)
```

- No external dependencies
- Data lost when the process exits
- Best for short-lived scripts and development

### Redis

Uses `aiocache` with the `REDIS` backend and `PickleSerializer`.

```python
config = CacheConfig(
    backend="redis",
    ttl=600,
    redis_url="redis://localhost:6379/0",
)
```

- Persistent across process restarts
- Shared across multiple processes
- Requires a running Redis server

## TTL (Time-To-Live)

Different data types use different TTL values:

| Data Type | Default TTL | Notes |
|-----------|-------------|-------|
| Current quotes | 3600s (1h) | Via repository payload |
| Historical data | 3600s (1h) | Rarely changes |
| Company info | 3600s (1h) | Fairly stable |
| Company summary | 86400s (1d) | Wikipedia summary |
| Market depth | 60s | Changes frequently |
| News | 1800s (30m) | Moderate frequency |
| Sector list | 86400s (1d) | Very stable |
| T-Bond info | 86400s (1d) | Updated infrequently |

The global default TTL is configured via `CacheConfig.ttl` (300 seconds). Individual repositories override this per-request via the `ttl` field in `RequestPayload`.

## Disabling Cache

```python
from bdfinance import BDStockClient, ClientConfig

config = ClientConfig(enable_cache=False)
async with BDStockClient(config=config) as client:
    # No caching — every call hits the network
    ...
```

Or via environment variable:

```bash
export BDFINANCE_ENABLE_CACHE=false
```

---

## Clearing Cache

```python
async with BDStockClient() as client:
    await client.clear_cache()
```

---

## Custom Serialization

The `CacheManager` handles non-standard types via a custom `default()` JSON serializer used during key generation:

| Type | Serialization |
|------|---------------|
| Objects with `.isoformat()` | ISO format string |
| Pydantic models | `.model_dump()` dict |
| `set` / `frozenset` | Converted to list |
| Objects with `__dict__` | Dict representation |
